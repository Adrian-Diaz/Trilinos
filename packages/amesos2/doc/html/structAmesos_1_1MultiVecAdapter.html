Detailed Description

template<class MultiVecType>
struct Amesos::MultiVecAdapter< MultiVecType >

A templated MultiVec class adapter for Amesos2.

Specializations of this templated class provide a unified interface to MultiVec types for Amesos2. Any specializations are expected to implement the following methods:


Implementation Requirements:

  • Default constructor 

     MultiVecAdapter<MultiVecType>();

  • Wrapper constructor 

     MultiVecAdapter<MultiVecType>(MultiVecType* const mat);

  • Copy constructor 

     MultiVecAdapter<MultiVecType>(MultiVecAdapter<MultiVecType>& const);

  • MultiVec scaling operation. Scales each element by alpha 

     void scale(const scalar_type alpha);

  • MultiVec add operation. Replace each element in this with 

     void add(const scalar_type beta, MultiVecAdapter<MultiVecType> B, const scalar_type alpha);

  • Method to get locality of multivec, either globally or locally indexed 

     bool isLocal()

  • Method to get multivec communicator 

     virtual const Teuchos::RCP<const Teuchos::Comm<int> >& getComm() const;

  • Methods to get the local and global length of vectors and number of vectors 

     local_ordinal_type  getLocalLength();
       local_ordinal_type  getLocalNumVectors();
       global_ordinal_type getGlobalLength();
  global_ordinal_type getGlobalNumVectors();

  • Method to access number of nonzero entries 

     local_ordinal_type  getLocalNNZ();
       global_ordinal_type getGlobalNNZ();

  • Method to get the maximum number of nonzeros in all rows. 

     global_ordinal_type getMaxNNZ();

  • Map methods. We should make this method more general, in that we should not be constraining ourselves to a particular Map object, and I don't believe they all inherit from the same base class, and they all have different interfaces. Is there some way that we can get the functionality from them, but make the interface more abstract? 

     Teuchos::RCP<Map> getRowMap();
       Teuchos::RCP<Map> getRangeMap();
  Teuchos::RCP<Map> getDomainMap();

  • Row access methods 

     void getLocalRowCopy(local_ordinal_type localRow,
                            const Teuchos::ArrayView<local_ordinal_type> &indices,
                            const Teuchos::ArrayView<scalar_type> &values,
                            size_t &numEntries) const;
 void getGlobalRowCopy(global_ordinal_type globalRow,
                       const Teuchos::ArrayView<global_ordinal_type> &indices,
                       const Teuchos::ArrayView<scalar_type> &values,
                       size_t &numEntries) const;

  • Data access methods 

     void get1dCopy(scalar_type* A, int lda) const;
       void get1dView(scalar_type** A, int* lda) const;
       void get2dCopy(scalar_type** A, int lda) const;
       void get2dView(scalar_type*** A, int* lda) const;

  • Method to convert MultiVec to serial. Amesos2 itself should not have to deal with the Maps (Epetra_Map, Tpetra::Map, etc) for this to work, the method should simply return a reference to a serial *copy* of the original multivector.

     RCP<multivec_type>& serial();

  • Method to revert MultiVec back to distributed. Should reverse the changes made to any method which localizes the MultiVec.

  • Get a description of this adapter 

     std::string description();

  • Print the multivec to the os output stream 
     void describe(Teuchos::FancyOStream& os) const;
